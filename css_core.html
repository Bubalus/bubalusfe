BubalusFE Web 前端開發工具集 —— 沒錯，雖然有那麽多英文，但這并不是什么高級貨！

BubalusFE，是一組就像遛狗大嬸一樣常見的 WEB 前端開發工具集，它所提供的功能也好像總是色迷迷盯著遛狗大嬸的散步大爺一樣普通，蹲在路邊虐殺螞蟻的小女孩也說：“它根本沒什麽了不起！”

它誕生于2007年的秋天，一個懶惰的傢伙爲了減輕工作壓力而拼湊出了它的原型。這個傢伙真是個會耍小聰明的懶鬼，使用這組工具后，他的工作用時節省了一半，但他還是向領導上報跟以前一樣多的工時，從此他感覺工作變得輕鬆了許多。真是可喜可賀！可喜可賀！

CSS 重置 用於統一各瀏覽器下的呈現

一個可配置的 CSS 重置工具 assets/css/bubaluslayout/core/reset.scss

包含「基礎」和「擴展」兩種重置類型。其中「基礎」重置類型大量參考了大名鼎鼎的 normalize.css，而「擴展」類型則在「基礎」類型的基礎上加強了呈現效果，尤其是表單和排版的方面。 

// assets/css/bubaluslayout/_config.scss

// 只开启基本重置
$only-base-reset: 		false !default;

IE-Hack 用於對早期版本的IE瀏覽器提供

Hack 的實現

BubalusFE的模板文件中，HTML 標籤被插入了一些瀏覽器識別信息，如 lt-ie7 等，所以，大部分情況下本工具不會使用傳統的 hack 選擇器，而會根據 HTML 標籤中的信息來進行匹配。
而在一些 Javascript 組件中，IE-Hack 則更多依賴 Modernizr.js 提供的功能。 

<!--[if lt IE 7]>      <html class="no-js lt-ie9 lt-ie8 lt-ie7"> <![endif]-->
<!--[if IE 7]>         <html class="no-js lt-ie9 lt-ie8"> <![endif]-->
<!--[if IE 8]>         <html class="no-js lt-ie9"> <![endif]-->
<!--[if gt IE 8]><!--> <html class="no-js"> <!--<![endif]-->

核心 Hack assets/css/bubaluslayout/core/iehacks.scss

包含了一些全局 Hack 和一些用於解決核心 CSS 組件在早期 IE 版本中出現的問題的 Hack。
再重申一下：本工具箱只對早期版本 IE 提供有限的支持，因為作者是「沒必要保證一致，不影響使用就好」的那一派，而且過多使用 Hack 只會更加加重 早期 IE 的渲染負擔，進而拖累早期 IE 本不來就怎樣的性能。

響應系統 爲響應式應用提供基礎

通過媒體查詢 @media 來對各種分辨率的屏幕或打印機提供專有的樣式。

響應式的應用

BubalusFE 核心 CSS 提供了一套響應式的柵格系統和一組響應式顯隱的工具類，附帶的一些 CSS 模塊也都具備響應式效果。
他們可以在 BubalusFE 預置的四種常見響應設置（ 手機、 平板、 PC、打印機 ）中實現各自專有的響應式效果

編寫自己的響應式設計

CSS 配置文件 assets\css\config 中提供了一些預置的方法和響應點變量，利用這些方法和變量來編寫自己的響應式設計。
這些内容應該寫到 assets\css\project\responsive.scss 裏面。

  • $screen-phone: 480px
    - 通常为智能手機竪屏分辨率
  • $screen-tablet: 768px
    - 通常为平板电脑竖屏分辨率
  • $screen-desktop: 1024px
    - 通常为平板电脑横屏分辨率
  • $screen-large-desktop: 1200px
    - 通常作为笔记本电脑和台式电脑屏幕分辨率分界点
  • @mixin screen($res-min, $res-max)
    - 寬度區間內觸發響應
  • @mixin max-screen($res)
    - 最大寬度以內觸發響應
  • @mixin min-screen($res)
    - 最小寬度以內觸發響應
  • @mixin screen-height($res-min, $res-max)
    - 高度區間內觸發響應
  • @mixin max-screen-height($res)
    - 最大高度值以內觸發響應
  • @mixin min-screen-height($res)
    - 最小高度值以內觸發響應
  • @mixin hdpi
    - 高清屏移動設備專用
  • @mixin ipad
    - ipad專用
  • @mixin ipad-retina
    - 配備Retina顯示屏的ipad專用
@include screen(768px, 1280px) { ... }

// 使用上面的方法將會生成

@media screen and ( min-width: 768px ) and ( max-width: 1280px ) {
  ...
}

============================

@include min-screen($screen-phone)  { ... }

// 使用上面的方法將會生成

@media screen and ( min-width: 480px ) {
  ...
}

============================

@include max-screen-height(768px)  { ... }

// 使用上面的方法將會生成

@media screen and ( max-height: 768px ) {
  ...
}

============================

// 如果是針對高清屏移動設備的項目，可以直接使用下面的方法實現橫豎屏的響應

@include hdpi { ... } 
@include hdpi(2) { ... } // 視網膜屏

============================

// 如果是針對ipad的項目，可以直接使用下面的方法實現橫豎屏的響應

@include ipad { ... }
@include ipad-retina { ... }  // 針對Retina顯示屏

柵格系統

12列響應式柵格系統，可實現多種形式的響應式佈局

柵格的應用

BubalusFE 的柵格系統具備響應式效果，而這種響應式效果是依賴於三種不同的類聲明來實現的。

  • .col-[i] 該類代表一般柵格，不具備響應式效果，始終保持佈局。
  • .col-table-[i] 該類為針對平板電腦的響應式柵格，在瀏覽器寬度小於 768px時佈局會崩壞，每個柵格獨佔一行。
  • .col-desktop-[i]該類為針對桌面顯示器的響應式柵格，在瀏覽器寬度小於 1024px時佈局會崩壞，每個柵格獨佔一行。
